# Multi-Company / Watchlist — Design

How the system tracks many companies, each with statements accumulating over periods.
Mostly supported by the existing data model; this captures the organization + workflow decisions.

## Decisions (2026-06-23)
- **Watchlist = flat list.** One list of all tracked companies, no themed groups. (Grouping can come later.)
- **Entry = manual add first.** You add a ticker to the watchlist, THEN upload its BCTC. Deliberate, not auto-created on upload.
- **History = keep everything.** Every ingested statement stays forever (storage is tiny — text + SQLite).
  Best for long-term trend / CAGR.
- **Cross-company = deferred.** v1 analyzes one company at a time. Compare/rank across watchlist comes later.
- **Company hub note = auto-refreshed index + trend.** Hermes regenerates `<SYM>.md` on each new statement:
  links to all statement/report notes + a ratio-trend table across periods. Single entry point per company.

## How it maps to existing structures
**SQLite (already in `models/core.py`):**
- `instruments` = the watchlist (one row per tracked company). Manual add → insert a row.
- `documents` → `statements` (kind, period, fiscal_year) → `line_items`. Multiple statements per company over
  time = period history. "Keep everything" = never delete old statement rows.

**Obsidian vault:**
```
Companies/
└── VNM/
    ├── VNM.md                       ← hub (MOC): auto index + ratio-trend table across periods
    ├── Statements/
    │   ├── VNM CĐKT Q4-2024.md
    │   ├── VNM CĐKT Q3-2024.md      ← history accumulates
    │   └── VNM KQKD Q4-2024.md
    └── Reports/
        └── VNM Phân tích Q4-2024.md
```

**UI (Helix prototype):** has a flat watchlist store already; use it as the company list. Adding a ticker =
add instrument. (Themed groups in the prototype → ignore for v1, flat only.)

## Workflow
```
1. Add ticker (manual) → instrument row + Companies/<SYM>/ folder + empty hub note
2. Upload BCTC for that ticker → documents → OCR → statements + line_items → reconcile
3. Proactive: save_statement_note (per statement) → Hermes analyze → save_report
4. Hermes regenerates <SYM>.md hub: re-index notes + rebuild ratio-trend table across all periods
5. (later) cross-company: compare/rank across the flat watchlist
```

## Company hub note (`<SYM>.md`) content
- Frontmatter: `symbol, name, market, tracked_since, tags`.
- **Index:** wikilinks to every statement note + report note, grouped by period (newest first).
- **Ratio-trend table:** key ratios (biên lãi gộp/ròng, ROE, ROA, nợ/vốn chủ, thanh khoản) across all periods
  — one column per period, so trends are visible at a glance. Built from `compute_ratios` over each period's statement.
- Regenerated by a Hermes/MCP tool (`rebuild_company_hub(symbol)`) on each new statement.

## New MCP tool implied
- `rebuild_company_hub(symbol)` — re-index notes + recompute the cross-period ratio trend → write `<SYM>.md`.
  (Adds to the obsidian tool group in `HERMES_BRAIN.md`.)

## Out of scope (v1)
Themed watchlist groups, cross-company comparison/ranking, auto-create-on-upload, rolling-window pruning.
